# Day 1 – Hour 4 (1:00 AM – 2:00 AM)

## Objective
To test and improve the performance of the voice-controlled prototype.


## Work Done
- Tested all voice commands multiple times
- Verified ON/OFF control of 3 LEDs
- Added wake-up and sleep commands for power saving
- Checked system response time and accuracy
- Improved voice clarity and distance testing

## Testing Observations
- System works accurately for predefined commands
- Best performance observed at close distance (0–1m to 0-5m)
- Wake-up command activates system before execution
- Sleep command reduces unnecessary operation

## Improvements
- Implemented wake-up command to activate system
- Implemented sleep command to save power
- Ensured proper wiring connections
- Optimized command speaking clarity
- Stabilized power supply

## Learning
- Voice recognition depends on environment and clarity
- Wake-up mechanism improves system efficiency
- Power saving features are important in embedded systems
- Hardware stability affects system performance
